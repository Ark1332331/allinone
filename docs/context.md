# 项目上下文

> 跨会话共享上下文。当前执行状态见 `docs/active/current-focus.md`。

## 项目定位

allinone — 个性化学习助手。从"通用 repo wiki 工具"转型为"陪伴用户理解课件的个人助手"。

## 用户

南京大学大一，智能科学与技术专业。
- 数学课件 PDF 划线提问是最紧迫需求
- 其次是专业课 PPT 复习突击
- 远期：强化学习/科研方向的材料阅读

## AI 角色定义

有判断力的思维伙伴。用户原话：
> "你不是一个无能听从我想法的 agent，而是在更高知识储备下建议、辅助、甚至纠正批评的 AI。"

触发条件：
- 规划结束时自问：有没有他应该考虑但还没提出的东西？
- 发现更优解时直接给出，不只列选项
- 反复讨论同一类问题时，指出底层能力缺口

## 已对齐的产品决策

- 主路径：学科 → 资料架 → 导入 → 阅读 → 划线提问 → 连续追问
- 划线提问是通用交互层，作用于原文、md、笔记、LLM 回复
- 数学 PDF 不能只靠 markdown 转写，需要源文件阅读层+截图提问
- 用户偏好精简 UI，讨厌大框/拖拽/多步骤
- pre-assessment 是阅读页的一个按钮，不是独立入口
- 不做：通用 wiki、复杂表单、跨资料拼接、自动画像写回

## 技术栈

- 前端: Next.js 15 + React 19 + Tailwind CSS 4 + pdfjs-dist
- 后端: copilot-api（LLM 代理，DeepSeek/OpenAI 兼容）
- 存储: 浏览器 localStorage
