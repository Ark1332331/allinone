# 阅读交互修复计划

> 状态：重写版。上一版计划把目标写成了“打开右上当前 Q&A 大面板”，这与用户真实需求不一致，导致实现继续偏。后续执行必须以本计划为准。

## 真实目标

阅读器要支持“边读边问”，不是把问题送进一个历史页或大侧栏。

用户在 PDF、结构化 Markdown、LLM 回答里选中内容时，应该在当前位置附近出现一个小提问框。提交后回答应该出现在这个阅读过程中的轻量浮窗里；锚点回看也应该回到当时那类小浮窗，而不是跳进完整历史页面或大面板。

## 不可妥协规则

1. 选中文字后，提问输入框必须出现在选区附近，用户不能去页面顶部或底部找入口。
2. LLM 回答里的文字必须能选中并提问，流程和 Markdown 正文一致。
3. 提交问题后，回答在当前轻量浮窗里展示；不要切换到完整历史页，不要打开占阅读空间的大侧栏。
4. PDF 截图提问保持右上角截图小框体验；点击 PDF 锚点时应恢复/打开对应截图问答小框，而不是外层大 Q&A 面板。
5. 结构化 Markdown 和 LLM 回答的文本锚点也应打开对应轻量问答浮窗；大侧栏先不要作为默认交互。
6. 小浮窗必须有明显关闭按钮。
7. 小浮窗必须能由用户调整大小；如果浮窗有左侧截图/上下文预览，左侧区域也必须能拖动调整宽度或彻底收起。
8. 关闭只是隐藏；删除才移除锚点和对应问答记录。
9. 顶部历史按钮只作为索引入口；点击某条历史记录也应回到对应轻量浮窗。
10. UI 文案保持中文，不要把中文界面替换成英文。
11. 不允许出现 mojibake/乱码文案。

## 当前问题复盘

- 上一版计划把“current-anchor Q&A box”写成目标，实际导致实现继续打开右侧大面板。
- 只用了 CSS 原生 `resize`，用户不容易发现，也不能调整左侧截图/上下文区域。
- LLM 回答选区虽然写了 `data-selection-scope`，但实际验收仍失败，必须用浏览器行为验证，而不是只看代码。
- PDF 锚点点击目前从外层 `ReadingWorkspace` 打开大面板；应改为让 `PdfReadingSurface` 自己恢复对应截图问答小框。
- 部分新增文案变成英文，违背中文产品界面。

## 实现方案

### 任务 1：先清理错误方向

- [ ] 把 `ReadingWorkspace.tsx` 里新增的英文界面文案全部改回中文。
- [ ] `createSelectionSnippetDraft()` 的标签保持中文：
  - `回答片段 N`
  - `PDF 片段 N`
  - `片段 N`
- [ ] 不再把“右侧大 Q&A 面板”当作默认目标。
- [ ] 大侧栏可以暂时保留代码，但不能作为选区提问和锚点点击的默认展示。

验收：

- 页面上不出现 `Current Anchor`、`Segment Q&A`、`Ask now`、`Create snippet`、`All question history` 等英文阅读器文案。
- `node_modules\.bin\tsc.cmd --noEmit` 通过。

### 任务 2：建立统一轻量问答浮窗

在 `ReadingWorkspace.tsx` 中新增/整理一个文本问答浮窗状态，用于 Markdown 正文和 LLM 回答：

```ts
type FloatingThreadState = {
  snippetIndex: number;
  x: number;
  y: number;
  width: number;
  height: number;
  isOpen: boolean;
};
```

- [ ] 选区提交问题后，创建/定位 snippet，并打开 `FloatingThreadState`。
- [ ] 回答消息渲染在该浮窗内。
- [ ] 浮窗内继续追问仍写回同一个 snippet thread。
- [ ] 浮窗有关闭按钮。
- [ ] 浮窗有明显拖拽/调整大小手柄，不只依赖浏览器右下角 `resize`。
- [ ] 关闭不删除锚点；删除按钮才删除 snippet。

验收：

- Markdown 正文选中一段，提问框在选区附近。
- 提交后回答就在轻量浮窗里出现。
- 继续阅读时页面不跳转、不回到第一页、不打开大历史页。

### 任务 3：修 LLM 回答选区提问

- [ ] LLM 回答区域必须使用和 Markdown 正文同一套选区捕获逻辑。
- [ ] 不要只依赖局部 `onMouseUp`；保留 document-level pointerup，但要能识别当前 selection 是否落在 LLM 回答容器内。
- [ ] 选中 LLM 回答后，提问小框必须出现在选区附近。
- [ ] 提交后创建 `source: 'assistant_reply'` 的独立 snippet。
- [ ] 新回答显示在轻量浮窗，不显示在底部通用追问框，也不打开大侧栏。
- [ ] 已保存的回答选区在原回答中显示下划线/高亮锚点，点击锚点打开该轻量浮窗。

验收：

- 先问一个问题，等 LLM 回答出来。
- 在回答里框选任意一句，旁边出现提问框。
- 输入自定义问题并提交。
- 新回答出现在当前轻量浮窗。
- 原回答中留下锚点；点击该锚点只恢复这个问答浮窗。

### 任务 4：PDF 锚点回到截图小框

PDF 的截图问答已经更接近用户想要的形式，后续不要把它改回外层大面板。

- [ ] `PdfReadingSurface` 接收 snippets 时，能根据 snippet index 找到对应截图 snippet 和 messages。
- [ ] 点击 PDF 页上的锚点时，不调用外层大侧栏默认路径；改为在 `PdfReadingSurface` 内打开/恢复截图问答小框。
- [ ] 小框展示：
  - 已框选页码
  - 可收起/展开的截图预览和原问题
  - 当前回答
  - 继续追问入口
- [ ] 如果 snippet 有 `imageDataUrl`，恢复小框时使用已保存图片，不需要重新截图。
- [ ] 如果 snippet 没有消息，不显示锚点。

验收：

- 点击 PDF 上已有问答锚点，出现当时截图提问那种小框。
- 不进入完整历史页。
- 不打开外层大 Q&A 面板。
- 关闭小框后锚点保留。
- 删除后锚点消失。

### 任务 5：左侧截图/上下文可调

这条专门回应“左边也可以调”。

- [ ] 截图问答小框不要固定 `md:grid-cols-[10rem_1fr]`。
- [ ] 增加 `contextPaneWidth` 状态，例如默认 160px。
- [ ] 在截图预览和回答区之间增加可拖动竖向分隔条。
- [ ] 拖动分隔条可调整左侧截图/上下文宽度。
- [ ] 保留“收起截图和问题”按钮，收起时回答区占满。
- [ ] 小框整体也能调整宽高，且调整手柄明显。

验收：

- 用户可以拖动改变左侧截图预览宽度。
- 用户可以收起左侧上下文。
- 用户可以调整整个浮窗大小。

### 任务 6：历史页只做索引

- [ ] 顶部历史按钮可以打开索引页。
- [ ] 索引页点击某条记录时，关闭历史索引，打开对应轻量浮窗。
- [ ] 索引页不要成为锚点点击的默认落点。

验收：

- 只有主动点顶部历史按钮才出现完整历史索引。
- 从索引点记录，也回到对应轻量浮窗。

- [ ] LLM 回答选区提问框在选区附近。
- [ ] LLM 回答选区提问后出现新轻量浮窗和锚点。

- [ ] 小框整体可调大小。

## 禁止事项

- 不要再把“右上当前 Q&A 大面板”当成这次修复完成标准。
- 不要把用户要求翻译成英文 UI。
- 不要只靠 `tsc` 就宣称 LLM 回答选区已修好。
- 不要让锚点点击进入完整历史页。
- 不要在 PDF 截图问答已经接近目标时，把它改成外层通用面板。
